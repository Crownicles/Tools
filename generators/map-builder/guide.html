🗺️ Map Builder — Mode d'emploi

← Retour à l'outil

Le Map Builder est un éditeur visuel pour positionner les marqueurs (POIs et arêtes) des cartes de Crownicles sur leurs fonds respectifs, puis pousser le résultat sous forme de Pull Request sur les dépôts du projet.

1. Concepts

Crownicles découpe son univers en mapLocations (POIs, ex: ville, caverne) et mapLinks (les chemins entre POIs). Ces données vivent dans Crownicles/Crownicles/Core/resources/{mapLocations,mapLinks}/.

Chaque POI possède un attribut (continent1, haunted, pve_island_entry, etc.) qui détermine sur quel fond de carte il s'affiche. Une mapPage regroupe un fond + ses POIs/arêtes via un fichier JSON dans Crownicles/Tools/generators/map-builder/mapCoords/ : 

{
  "mapPage": "oceanic_exterior",
  "displayName": "Île océanique - Extérieur",
  "includeAttributes": ["pve_island_entry", "pve_island"],
  "idRange": {"min": 1200, "max": 1299},
  "backgrounds": { "fr": {"filename": "oceanic_exterior_fr.jpg", "width": 2048, "height": 1555}, ... },
  "marker": { "image": "cross.png", "size": {...} },
  "nodes": { "1200": {"x": 950, "y": 740}, ... },
  "edges": { "1200_1201": {"startMap": 1200, "endMap": 1201, "x": 1100, "y": 800}, ... }
}

L'outil = un éditeur visuel pour le bloc nodes + edges, plus un workflow PR.

2. Créer un PAT GitHub

Le PAT (Personal Access Token) est obligatoire pour pousser des PRs. Sans PAT, l'outil reste en lecture seule (Viewer) — tu peux quand même charger et regarder.

Option A : token classique (le plus simple)

  1. Va sur github.com/settings/tokens/new.
  2. Note : Crownicles Map Builder.
  3. Expiration : 90 jours (ou plus selon tes préférences).
  4. Scopes à cocher :
    • repo (tout le bloc) — pour créer branche, pousser fichiers, ouvrir PRs.
    • workflow — optionnel mais utile si tu touches un .github/workflows.
  5. Clique Generate token, copie le ghp_... immédiatement (visible une seule fois).
  6. Reviens dans le Map Builder, colle-le dans le champ PAT GitHub, clique Sauver token. Le token est stocké en localStorage de ton navigateur uniquement.

Option B : token fine-grained (recommandé si tu veux limiter)

  1. Va sur github.com/settings/personal-access-tokens/new.
  2. Resource owner : Crownicles (ou ton compte si fork).
  3. Repository access : Only select repositories → coche Crownicles/Tools et Crownicles/Website.
  4. Repository permissions :
    • Contents : Read and write
    • Pull requests : Read and write
    • Metadata : Read (auto)
  5. Generate, copie, colle dans l'outil, Sauver.

Option C : via la CLI gh (devs)

gh auth login           # si pas déjà fait
gh auth token           # affiche le token actuel
# Copie la valeur gho_... → champ PAT GitHub → Sauver

Le token gho_ du CLI possède déjà repo, workflow, read:org, gist par défaut.

⚠️ Le PAT est secret. Ne le commit jamais, ne le partage pas. Pour le révoquer : github.com/settings/tokens → Delete.

3. Workflow type

  1. Ouvre l'outil, clique Charger (récupère mapPages depuis Crownicles/Tools + mapLocations/Links depuis Crownicles/Crownicles).
  2. Sélectionne l'onglet de la carte à éditer (Continent, Île gelée, etc.).
  3. Passe en mode Editor (radio en haut à gauche).
  4. Déplie Coords manquantes à droite → glisse-dépose chaque ligne sur l'image. Les nœuds (N) deviennent des croix, les arêtes (E) deviennent un point sur la ligne entre 2 nœuds.
  5. Ajuste les positions en cliquant + draggant les croix sur la carte.
  6. Quand tu es content, clique PR Tools (coords) → l'outil crée une branche map-update-* sur Crownicles/Tools et ouvre la PR.
  7. Optionnel : clique Rendu images… pour générer les .webp avec croix superposées, puis PR Website (images) pour les pousser sur Crownicles/Website.

4. Interface — qui fait quoi

BoutonEffet
ChargerRe-fetch tout depuis GitHub (mapCoords + mapLocations + mapLinks + place names).
Sync ← CrowniclesDétecte les POIs/attributs qui ne sont pas couverts par les mapPages existantes et propose : nouvelle mapPage, nouvelle île (bloc d'ID distinct), ou extension d'idRange.
PR Tools (coords)Push les mapCoords/*.json modifiés sur Crownicles/Tools via une nouvelle branche + PR.
Rendu images…Génère des .webp avec les croix dessinées sur chaque fond, pour chaque langue.
PR Website (images)Push les images générées sur Crownicles/Website dans public/ressources/maps/cursed/.
Export coords ZIPTélécharge un ZIP de tous les mapCoords/*.json (utile pour offline ou backup).
Mode Viewer/EditorViewer = lecture seule. Editor = drag-drop activé.
FR / ENBascule le fond de carte selon la langue (si une image EN existe).
Dépôts avancésPermet de pointer vers un fork (ex: monuser/Crownicles) au lieu du repo officiel.

Panneau de droite :

  • Synchronisés : POIs dont la position est déjà enregistrée.
  • Coords manquantes : POIs reconnus par Crownicles mais non placés (à drag-drop).
  • Orphelins : POIs présents dans le JSON local mais qui n'existent plus côté Crownicles (à nettoyer).

5. Ajouter une nouvelle île / carte

  1. Demande au team game de t'envoyer l'image (idéalement déjà nommée selon la convention nom_FR.jpg + nom_EN.jpg).
  2. Place-la dans Crownicles/Tools/generators/Ressources/ ET Crownicles/Website/public/ressources/maps/.
  3. Ajoute un fichier mapCoords/<nom>.json sur Crownicles/Tools (voir exemple en section 1). Le moyen le plus simple : copie oceanic_exterior.json, change les valeurs.
  4. Ou utilise le bouton + Nouvelle page dans l'outil pour créer une mapPage à la volée (mais tu devras quand même pousser l'image manuellement).
  5. Le bouton Sync ← Crownicles détecte automatiquement les POIs avec un nouvel attribut ou un bloc d'ID non couvert et propose de créer la mapPage stub pour toi.

6. Ajouter / changer un fond de carte

L'outil cherche les fonds dans cet ordre :

  1. https://raw.githubusercontent.com/Crownicles/Tools/master/generators/Ressources/<filename>
  2. https://raw.githubusercontent.com/Crownicles/Website/<branch>/public/ressources/maps/<filename>

Le filename et les dimensions doivent matcher ce que tu déclares dans backgrounds du JSON. Si tu remplaces une image par une plus grande, n'oublie pas de mettre à jour width/height sinon les nœuds existants se retrouvent décalés.

8. Dépannage

« Coords manquantes (0) » alors qu'on attend des POIs

  • As-tu cliqué Charger au moins une fois ?
  • L'onglet sélectionné est-il bien la bonne mapPage (chaque île a son propre onglet) ?
  • Vérifie dans la console DevTools : state.locations contient-il les IDs attendus ?
  • Le panneau peut être replié — clique dessus pour déplier.

« Erreur 401 / 403 » lors d'une PR

  • PAT manquant, expiré, ou sans le scope repo. Régénère-le.
  • En fine-grained, vérifie que tu as bien sélectionné les repos cibles.

« Limite GitHub atteinte » (rate limit)

  • Sans PAT : 60 req/h. Avec PAT : 5000 req/h.
  • Sauve un PAT même temporaire pour débloquer.

Les croix ne sont pas drag-droppables

  • Tu es en mode Viewer. Passe en Editor.

Le fond de carte ne s'affiche pas (rectangle vide)

  • Le fichier image n'existe pas aux URLs attendues. Vérifie les 2 candidats (Tools/Ressources puis Website/public/ressources/maps).
  • Le filename dans le JSON ne matche pas le nom réel (case-sensitive).

Une PR a été créée mais je veux annuler

  • Ferme la PR sur GitHub et supprime la branche map-update-*. Aucun side-effect sur les autres branches.
